# D2 今日任务说明

## 一句话目标

从一个空文件夹开始,亲手搭出一套**企业级、可安装、可移植的项目初始化系统**——最终能在命令行直接敲 `odp-init` 一键初始化整个项目。

---

## 你要交付什么

完成后你的仓库应包含以下文件,且能通过所有检查点:

```
ODPlatform/
├── .odp-workspace                                  ← 项目根标记文件
├── .gitignore

├── apps/
│   └── platform/
│       ├── pyproject.toml                          ← 可安装包配置(hatchling)
│       ├── README.md
│       └── src/odp_platform/
│           ├── __init__.py
│           ├── _version.py
│           ├── common/
│           │   ├── __init__.py
│           │   ├── paths.py                        ← 路径常量
│           │   ├── logging_utils.py                ← 彩色日志
│           │   ├── string_utils.py                 ← 中英文对齐
```

外加各占位目录的 README(`apps/web-backend` 等)。

## 验收标准(自查)

做完后逐项确认:

1. **commit 数正确**:`git log --oneline` 恰好 **11 次** commit,每条 message 都能用一句话说清解决了什么。
2. **`odp-init` 命令可用**:
   ```
   odp-init
   ```
   能跑通,输出包含:环境快照、彩色日志、对齐的目录汇总表、性能报告。
3. **目录被正确创建**:运行后 `data/`、`models/`、`runs/`、`apps/platform/configs` 等 18 个目录存在。
4. **日志文件落盘**:`apps/platform/logging/init_project/` 下生成了 `.log` 文件。
5. **三种调用方式都能跑**:
   - `odp-init`
   - `python -m odp_platform.cli.init_project`
   - `python scripts/init_project.py`
6. **源码树干净**:`apps/platform/src/` 下只有 `odp_platform/` 一个目录,**没有 `*.egg-info/`**。
7. **每个模块单测能跑**:每个 common 模块单独 `python xxx.py` 都有正确自测输出。

## 关键概念检查(理解,不是背诵)

跑完后,用自己的话回答这几个问题(可写一页反思):

1. 为什么 `paths.py` 用 marker file 定位项目根,而不是 `parent.parent.parent`?
2. `ROOT_DIR` 和 `APP_DIR` 的区别是什么?为什么 `data/` 放在 `ROOT_DIR`,`logging/` 放在 `APP_DIR`?
3. 业务模块为什么只写 `logger = logging.getLogger(__name__)`,不自己配 handler?handler 是谁、在哪里、配几次?
4. 为什么含中文的表格不能直接用 Python 的 `<25` 对齐?
5. 为什么选 hatchling 而不是 setuptools?`*.egg-info/` 问题是谁的问题?
6. `[project]` / `[project.scripts]` 和 `[tool.hatch.*]` 有什么本质区别?换 backend 时哪部分要改?
7. 既然装完包能 `python -m odp_platform.cli.init_project`,为什么还要 `odp-init` 这个 entry-point?

## 环境准备(开工前)

```
python --version          # 需 ≥ 3.10
conda create -n odplat python=3.12
conda activate odplat
git --version
```

**Windows 用户**:强烈建议装 [Git for Windows](https://git-scm.com/download/win),用其自带 **Git Bash**,这样所有命令与 Linux 一致,VSCode 默认终端也可切到 Git Bash。

---

## 注意事项

- **创建文件优先用编辑器**(VSCode/PyCharm),保存编码统一选 **UTF-8 无 BOM**;编辑器会自动创建沿途目录。
- **每完成一个阶段立即 commit**,不要攒着一次性提交。
- 在**仓库根目录**执行所有命令(除非特别说明)。
- 阶段 7 之前运行需 import 项目代码的命令要带 `PYTHONPATH=apps/platform/src`;装包后这一坨彻底不用了。
- `torch` + `ultralytics` 较大(2-4 GB),网络慢可先跳过,`system_utils` 会自动降级,不影响主流程跑通。
