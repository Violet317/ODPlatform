# D3 今日任务说明

## 一句话目标

在已完成 D2 + D2.5 的仓库上,搭出一个完整的数据准备子系统 `data_pipeline/`,最终跑通一行命令:

```bash
odp-transform --dataset <名字> --format pascal_voc
```

→ 原始数据集自动转 YOLO 格式 + 划分 train/val/test + 生成 ultralytics 训练 yaml。

## 验收标准(自查)

1. **commit 数正确**:`git log --oneline` 看到本文 **10 次** commit。
2. **注册表生效**:
   ```bash
   python -c "from odp_platform.data_pipeline import list_capabilities; print(list_capabilities())"
   ```
   输出 `{'pascal_voc': ('detect',), 'coco': ('detect', 'segment'), 'yolo': ('detect',)}`
3. **单元测试全过**:`cd apps/platform && pytest tests/ -v` → 16 个测试通过(splitter 8 + registry 6 + e2e 2)。
4. **CLI 跑通**:对一个 ≥50% 覆盖率的 VOC 数据集跑 `odp-transform --dataset X --format pascal_voc`,退出码 0,生成 `apps/platform/configs/datasets/X.yaml`。
5. **yaml 含 odp_meta**:`cat` 那个 yaml,能看到 `odp_meta` 块里有 `random_state`、`created_at`、`split.counts`。
6. **`--help` 能力矩阵**:`odp-transform --help` 末尾有从 registry 实时读出的格式能力矩阵。
7. **覆盖率 fail-fast**:残缺数据集(覆盖率 <50%)跑 `odp-transform` 立刻报 `ValueError`,不进入转换/训练。
8. **三个新命令并存**:`odp-init`、`odp-reset`、`odp-transform` 都还能用。

## 关键概念检查(理解,不是背诵)

跑完后用自己的话回答(写一页反思):

1. 为什么用注册表而不是工厂或抽象基类?三者分别解决什么不同的问题?
2. 为什么 `ConvertOptions.classes` 用 `Optional[List[str]] = None` 而不是 `= []`?`None` 和 `[]` 语义差别是什么?
3. COCO converter 的中转目录为什么必须用 `tempfile`,不能放 `data/raw/` 或项目根?
4. YOLO 已经是目标格式,为什么它的 converter 还要"写"到 `output_labels_dir`?"接口等价性"解决了什么?
5. `split_pairs` 里那 5 个边界判断和 `RATE_EPSILON` 是干嘛的?`1.0 - 0.7 - 0.3` 等于多少?
6. materializer 为什么用依赖注入接收目标目录,而不直接 import paths.py?举两个 DI 占优的场景。
7. orchestrator 里 `_user_classes` 和 `_final_classes` 为什么要分开两个属性?合并成一个会出什么 bug?
8. 覆盖率检查为什么要放在调 converter **之前**?如果不放,1000 图/50 标注的数据集会发生什么?
9. yaml 里的 `odp_meta` 块和 `schema_version: 1` 各有什么用?

## 注意事项

- **顺序很重要**:registry 的 `_lazy_init()` 会 import 三个 converter。阶段 1-3 期间**不要**跑任何触发 `get_converter()`/`list_capabilities()` 的代码(只测 `ConvertOptions()`),等阶段 4 三个 converter 都写完再统一验证注册表。**不需要"先注释掉两行 import"这种临时操作。**
- 改 `paths.py` 是**追加**到末尾 + 给 `get_dirs_to_initialize()` 加一行,不要重写整个文件。
- orchestrator **一次写最终版**(`_check_raw` 直接包含覆盖率检查),不写"第一版只检查目录"的过渡版本。
- 阶段 9 改完 pyproject 后必须 `pip install -e .` 重装,entry-point 才生效。
- 创建文件优先用编辑器,UTF-8 无 BOM。
- 在仓库根执行命令(除非命令里写了 `cd apps/platform`,记得用完 `cd -` 回来)。
